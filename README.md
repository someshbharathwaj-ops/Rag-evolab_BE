# RAG-evolab

A RAG (Retrieval Augmented Generation) application addon for the evolab to facilitate learners ecosystem in the evolab.

## Overview

This RAG application combines document ingestion, vector storage, and language model integration to provide contextual answers to user queries based on provided documents. The application features a Next.js frontend with a FastAPI backend that connects to the RAG processing pipeline.

## Prerequisites

- Python 3.8+
- Node.js 16+
- npm or yarn
- Weaviate instance (local or cloud)
- Ollama installed (for local LLM)

## Setup Instructions

### 1. Clone the Repository

```bash
git clone https://github.com/Akhils696/RAG-evolab.git
cd RAG-evolab
```

### 2. Backend Setup

#### Install Python Dependencies

```bash
cd backend
pip install -r requirements.txt
```

#### Set Up Environment Variables

Create a `.env` file in the root directory with the following variables:

```env
OPENAI_API_KEY=your_openai_api_key_here
WEAVIATE_URL=your_weaviate_instance_url
WEAVIATE_API_KEY=your_weaviate_api_key
MODEL_NAME=phi3:mini  # Using local model via Ollama
```

### 3. Frontend Setup

#### Navigate to Frontend Directory

```bash
cd frontend
```

#### Install Dependencies

```bash
npm install
```

### 4. Running the Application

#### Start the Ollama Service

```bash
# In a new terminal
ollama serve
```

#### Pull the Required Model

```bash
ollama pull phi3:mini
```

#### Start the FastAPI Backend

From the backend directory:

```bash
python api.py
```

#### Start the Next.js Frontend

In a new terminal, navigate to the frontend directory:

```bash
cd frontend
npm run dev
```

The application will be available at `http://localhost:3000`.

## Usage

1. Make sure your documents are processed and stored in the vector database
2. Start Ollama service with the required model
3. Start both the backend and frontend services
4. Access the frontend at `http://localhost:3000`
5. Enter your query in the text area and submit
6. View the contextual response generated by the RAG system

## API Endpoints

- `GET /` - API root endpoint
- `POST /query` - Submit a query to the RAG system
- `GET /health` - Health check endpoint

## Development

### Adding New Document Loaders

Document loaders are located in `backend/ingestion/loaders/`. Create a new loader by extending the base loader class.

### Customizing Prompts

Prompts are managed in `backend/rag/prompts.py`. Modify these to change how the LLM processes retrieved context.

### Changing Vector Store

Vector stores are implemented in `backend/vectorstores/`. You can swap implementations by modifying the retriever.

## Troubleshooting

- If you encounter issues with vector storage, verify your Weaviate connection and credentials
- Ensure Ollama is running with the required model pulled
- Check that both backend (port 8000) and frontend (port 3000) are running
- Make sure the NEXT_PUBLIC_BACKEND_URL in frontend/.env.local matches your backend server URL

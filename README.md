# RAG-Evolab

RAG-Evolab is a **Retrieval Augmented Generation (RAG) application** designed to support the Evolab learning ecosystem by enabling users to query documents and receive context-aware answers generated by a language model.

The system processes documents, stores their embeddings in a vector database, retrieves relevant information based on user queries, and generates responses using a language model.

---

# Features

* Document ingestion pipeline
* Vector storage using Weaviate
* Retrieval-based question answering
* Local LLM integration using Ollama
* Web interface for user queries

---

# Requirements

* Python 3.8+
* Node.js 16+
* npm
* Weaviate instance
* Ollama installed

---

# Installation

## Clone Repository

```bash
git clone https://github.com/Akhils696/RAG-evolab.git
cd RAG-evolab
```

---

# Backend Setup

Install dependencies:

```bash
cd backend
pip install -r requirements.txt
```

Create a `.env` file in the root directory:

```env
OPENAI_API_KEY=your_openai_api_key
WEAVIATE_URL=your_weaviate_url
WEAVIATE_API_KEY=your_weaviate_api_key
MODEL_NAME=phi3:mini
```

---

# Frontend Setup

```bash
cd frontend
npm install
```

---

# Running the Application

Start the Ollama service:

```bash
ollama serve
```

Pull the model:

```bash
ollama pull phi3:mini
```

Start the backend:

```bash
cd backend
python api.py
```

Start the frontend:

```bash
cd frontend
npm run dev
```

The application will be available at:

```
http://localhost:3000
```

---

# Usage

1. Add documents to the `backend/data/raw` directory.
2. Run the ingestion pipeline to process documents.
3. Start the backend and frontend services.
4. Access the application in the browser.
5. Enter queries related to the ingested documents.

---

# API Endpoints

GET `/`
Root endpoint.

POST `/query`
Submit a query to retrieve an answer from the RAG system.

GET `/health`
Backend health check.

---

# Notes

* Ensure the vector database is running before starting the backend.
* Ollama must be running with the required model available.
* The frontend expects the backend to run on port **8000**.
